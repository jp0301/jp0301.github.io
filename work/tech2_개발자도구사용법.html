Chrome 개발자 도구 사용법
6개월 전

크롬 개발자 도구는 chrome 크롬 → 설정 → 도구 더보기 → 개발자 도구(F12) 로 들어갈 수 있다.

개발자 도구를 이용하면 HTML, CSS, JavaScript 의 생산성을 극대화할 수 있다.

자주 쓰는 것들에는 Elements, Console, Source, Network, Application 이다.

Elements (요소)

  • 웹 페이지를 구성하고 있는 요소들을 의미한다.
  • 왼쪽 상단의 화살표를 클릭하면 요소들을 살펴볼 수 있다.
  • 오른쪽의 css 창에서 작대기 되있는 것은 우선순위에 밀려있는 것
  • Style은 우선순위에 의해 적용되지 않는 것들도 다보여주는 것
  • Computed 는 결과적으로 사용자에게 보이는 css를 나열해서 보여주는 것

aplication (어플리케이션)

  • 현재 웹페이지가 어떤 리소스로 구성되어있는가로 보여줌

source (소스)

  • 웹페이지에 삽입된 자바스크립트, css를 코드 레벨로 볼 수 있다.
  • 요소는 렌더링된 것을 보여주는 것이고
  • 실제로 페이지에 삽입된 코드 자체를 보여준다.

Network (네트워크)

  • 리소스를 가져오는 과정에서의 데이터 요청에 대한 정보를 가지고 있다.
  • 엘리멘트 탭에서 보이는 외부 리소스가 네트워크 통신으로부터 받아오는 것들을 의미한다.
  • FetchX/HR 을 한번 들어가보면 GET/ POST 방식
    • GET은 Request URL을 통해서 데이터를 요청하는 방식
    • url을 호출해서 url이 서버에 도달하면 원하는 데이터를 반환받는 방식
    • POST방식은 URL에 어떤 데이터가 들어가 있지는 않고
    • PAYLOAD를 통해서 데이터를 전달하게 되어 있다.

요약

  1. Elements : html/css 상태 확인
  2. Console : 콘솔(변수 정보를 확인하고 오류 메세지 표시)
  3. Sources : 스크립트 디버깅(break point 지정 및 변수 모니터링 등)
  4. Network : 브라우저에서 발생하는 통신 상태
  5. Performance(예전 Timeline 패널) : 성능 측정
  6. Memory(예전 Profiles 패널) : 메모리 사용 형태를 작성하고 누수 탐색
  7. Application(예전 Resources 패널) : 쿠키 및 스토리지 등의 내용 수집
  8. Security : Mixed content 이슈, 인증서 문제 등을 디버깅
