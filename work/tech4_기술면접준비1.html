JVM 메모리 구조

자바 프로그램은 자바프로그램 > JVM > OS > 하드웨어 순으로 실행되는데 프로그램이 실행되면 JVM은 OS로 부터 프로그램이 필요로 하는 메모리를 할당받고 JVM은 이 메모리를 용도에 따라 여러 영역으로 나누어 관리한다.

크게 Garbage Collector, Classloader, Runtime dataarea, Excution Engine

  • 가비지 컬렉터는 힙 메모리 영역에 생성된 객체들 중에서 참조되지 않은 객체를 탐색해서 제거한다.
  • 클래스로더는 클래스 파일을 로딩하여 검증하고 초기화한다.
  • 런타임 데이터에어리어는 클래스 파일을 저장하고
  • 익스큐션 엔진은 클래스 파일을 플랫폼에 맞는 기계어로 변환시켜 실행한다.

스레드의 생명주기

스레드 객체가 생성되면 생명주기를 갖게 된다. 크게 5가지로 나눌 수 있다.

  • New : 스레드가 만들어진 상태인 객체 생성
  • Runnable : 실행 대기 상태, 실행 상태로 언제든지 갈 수 있는 상태이다.
  • Running : 실행 상태,
  • Blocked : 일시 정지, 사용하고자 하는 객체의 LOCK이 풀릴 때까지 대기
  • Waiting : 일시 정지, 다른 스레드가 통지할 때까지 대기
  • Timed_waiting : 일시 정지, 주어진 시간 동안 대기
  • Terminated dead : 실행을 마친 상태(종료)

인터페이스란?

다른 클래스를 작성할 때 기본이 되는 틀을 제공하면서, 다른 클래스 사이의 중간 매개 역할까지 담당하는 일종의 추상 클래스를 의미한다. 추상메소드와 상수만을 포함할 수 있다.

다형성이란?

같은 메시지에 대해서 클래스에 따라 다른 행위를 하게 되는 특징이다. 같은 이름을 가지는 메서드에 대해서 인자 개수랑 데이터타입에 따라 수행되는 행위가 달라지는 것을 의미한다.

컬렉션 프레임워크란?

다수의 데이터를 쉽고 효과적으로 처리할 수 있는 표준화된 방법을 제공하는 클래스의 집합을 의미하며 즉, 데이터를 저장하는 자료 구조와 데이터를 처리하는 알고리즘을 구조화해서 클래스로 구현해 놓은 것이다.

Boxing과 UnBoxing?

원시 타입 : int, double, char, byte, short, float, long, boolean 참조 타입 : Integer, Character, Byte, Short, Float, Double, Long, boolean

같은 원시 타입을 참조 타입으로 변환하는 것을 의미한다. 언박싱은 참조 타입을 원시 탕븡로 변환한다. jdk 1.5부터는 오토 박싱, 오토 언박싱 기능이 가능해졌다.

스트림의 종류

바이트 입출력의 인풋 스트림, 아웃풋 스트림 문자 입출력의 Reader 와 Writer가 있다.

jdbc란?

자바에서 db 프로그래밍을 하기 위해 사용되는 api, 즉, 자바 언어와 db를 연결해주는 통로와 같은 것이다.

PreparedStatement 와 Statement의 차이는 무엇인가?

Statement는 정적 쿼리 시 사용하며 매번 파싱과정을 거쳐야 한다. 그래서 부하가 생길 수 있고 sql문을 명확히 알 수 있어 디버깅이 쉽다.

PreparedStatement는 동적 쿼리 시 사용하며 한번 파싱하면 그 동일한 sql 문장을 곧바로 파싱 과정 없이 Execution 할 수 있다. 반복적인 다량의 sql 수행 시 성능 상 이득이 있고 오류 발생 시 변수 입력 값을 알 수 없어 디버깅이 어렵다.
